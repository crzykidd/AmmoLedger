# ESPHome Configuration Examples

This folder contains [ESPHome](https://esphome.io) configurations for AmmoLedger-compatible scanner devices. Each example targets the canonical reference build described in [`../esp32-scanner.md`](../esp32-scanner.md) (Waveshare ESP32-S3-Touch-LCD-2 + Elechouse PN532 V3 over I2C).

**Validation status:** The examples in this folder are written against the reference hardware but await first-build validation. Specific timings, debounce behavior, and HTTP request shapes may be refined as real builds expose real issues. Each file's header comment documents its validation status.

## What's here

- [`intake-reader.yaml.example`](./intake-reader.yaml.example) — Moves a scanned box's location to a configured destination (e.g., "Range Bag"). Awaiting first-build validation.

## Planned configurations

- `intake-reader.yaml.example` — moves scanned boxes to a configured location.
- `bulk-programmer.yaml.example` — writes token URLs to blank NFC tags from desktop.
- `range-bag-reader.yaml.example` — scans attach expenditures to the current range session.
- `expend-logger.yaml.example` — fixed-quantity expenditure logger at the reloading bench.

## Customization

Each example requires customization before flashing:

- **AmmoLedger server URL** — set to your installation's HTTPS endpoint.
- **Device token** — generated by AmmoLedger when pairing a new device. See `../../prd/tagging.md` §7.5.
- **WiFi credentials** — your local network (use ESPHome `secrets.yaml` for these).
- **Action-specific parameters** — e.g., the target location ID for an intake reader.

## Contributing

**Improvements to a reference example.** If you flash one of these against the canonical hardware and find issues (wrong pins, missing debounce, broken HTTP shape, etc.), open a PR fixing the YAML in place. Update the header comment's validation status to reflect what you tested.

**New roles for the canonical hardware.** If you write a YAML for a scanner role not yet listed above, open a PR adding it. Follow the file-header convention used by existing examples.

**Configurations for community-validated alternative hardware.** If you build a working scanner using hardware different from the canonical reference (different ESP32 board, MFRC522 instead of PN532, different bus, etc.), drop your YAML here with a descriptive filename indicating the hardware (e.g., `intake-reader-mfrc522.yaml.example`), and add an entry to the "Community-validated hardware" section in [`../esp32-scanner.md`](../esp32-scanner.md).

When contributing any YAML:

- Use the `.yaml.example` extension to make clear the file requires customization.
- Include a file-header comment block documenting purpose, target hardware, validation status, and every customization point.
- Strip any real WiFi credentials, device tokens, or server URLs.
